# 功能性需求与用例描述模板

用于 3.3 分析模型（功能性需求）。

## 1) 需求条目表（先做）

| 需求编号 | 名称 | 参与者 | 说明 |
| -------- | ---- | ------ | ---- |
| FR-01    |      |        |      |
| FR-02    |      |        |      |

使用说明：

- 先列需求条目，再据此绘制用例图。
- 名称建议使用“动词+对象”结构。
- 说明写业务目标，不写实现细节。

## 2) 用例描述表（选核心用例）

| 字段       | 内容 |
| ---------- | ---- |
| 用例名称   |      |
| 用例描述   |      |
| 主执行者   |      |
| 优先级     |      |
| 触发条件   |      |
| 后置条件   |      |
| 基本事件流 |      |
| 异常事件流 |      |

使用说明：

- 仅展开核心或突破性业务用例，避免全量罗列。
- 基本事件流按步骤编号，异常事件流写清分支与处理策略。
- 推荐将该表放入对应业务系统小节内（如 3.3.1 订单子系统、3.3.2 库存子系统）”

## 3) 按业务系统展开（推荐骨架）

```text
3.3.x <业务系统名称>
- 子系统目标：
- 参与者与边界：
- 核心用例（1~2个）：
  - 用例A：<目标/触发条件/结果>
  - 用例B：<目标/触发条件/结果>
- 关键流程说明：
  - 结合活动图或时序图解释主流程与异常分支
```
