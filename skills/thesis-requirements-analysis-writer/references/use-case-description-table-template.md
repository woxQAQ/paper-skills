# 用例与需求表模板（3.3 功能性需求）

| 需求编号 | 名称 | 参与者 | 说明 |
| -------- | ---- | ------ | ---- |
| FR-01    |      |        |      |
| FR-02    |      |        |      |

使用要求：

- 必须先列该表，再给系统核心用例图。
- 名称建议使用“动词+对象”。
- 说明写业务目标，不写技术实现细节。

## 2) 系统核心用例图（必须出现）

使用要求：

- 明确系统边界与参与者。
- 图中被重点分析的用例，需在下文给出用例描述表。
- 正文只展开核心/突破性用例，其余可放附录。

## 4) 动态模型补充（按需）

- 活动图：用于描述流程控制与分支。

建议：动态模型只服务重点业务场景，不做形式化堆砌。
