# 技术选型质量检查清单

在输出最终文本前逐项判定是否通过。

## 聚焦检查

- [ ] 未复制通用框架介绍或百科式背景
- [ ] 章节重心是选型依据，而非技术词条堆砌
- [ ] 每个主要段落都与本项目场景直接相关

## 论证质量

- [ ] 论证链完整：需求 -> 准则 -> 对比 -> 最终选择
- [ ] 在信息允许时，至少讨论一个现实可行备选方案
- [ ] 所选技术能够对应并解决明确项目痛点
- [ ] 明确写出权衡与约束，不回避代价

## 工程落地性

- [ ] 预期收益具体可感知（如交付效率、维护成本、稳定性）
- [ ] 说明项目中的实际应用方式，不停留在抽象口号
- [ ] 语言简洁、技术化、不过度宣传

## 输出规则

若关键项未通过，先修订再交付终稿。
