# 系统测试质量检查清单

在输出终稿前逐项判定是否通过。

## 章节范围与定位

- [ ] 同时覆盖开发过程测试与产品测试。
- [ ] 测试范围、对象、环境、方法/工具、目标与验收基线声明清晰。
- [ ] 明确说明“详写什么、简写什么”的裁剪依据。

## 测试分析深度

- [ ] 同时说明显式与隐式测试需求获取过程。
- [ ] 需求来源可追溯到测试点。
- [ ] 测试层级选择基于风险与阶段，不是随意挑选。

## 用例设计质量

- [ ] 用例设计与所选测试需求一一对应。
- [ ] 关键测试至少使用两种不同用例设计方法。
- [ ] 通过/失败判据具体且可验证。

## 执行记录与度量

- [ ] 资源消耗有实际数据记录。
- [ ] 覆盖率分析能说明执行充分性。
- [ ] 报告缺陷统计、趋势解读、残留缺陷与未解决问题。
- [ ] 最终结论明确测试目标与通过准则是否达成。

## 写作完整性

- [ ] 不出现“全部通过且无问题”等无证据笼统结论。
- [ ] 数据与结论一致且可追溯。
- [ ] 压缩低价值、重复性测试项罗列。

## 输出规则

若关键项未通过，先修订并说明修复点，再交付终稿。
